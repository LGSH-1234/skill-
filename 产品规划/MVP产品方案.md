# MVP 产品方案

## 一、产品目标
打造一个面向泛用户的个人 Skill 展示网站，用于长期沉淀个人品牌、系统展示个人能力，并支持持续更新新的 Skill 与模板内容。

## 二、MVP 目标定义
虽然首版希望相对完整，但在功能优先级上仍需聚焦以下核心价值：
1. 清晰展示个人品牌形象
2. 结构化展示 Skill 内容
3. 支持 Skill 详情介绍与模板下载
4. 为后续持续新增内容提供稳定结构

## 三、目标用户
### 核心用户
- 对个人能力、AI 能力、方法论内容感兴趣的泛用户

### 用户诉求
- 快速理解你擅长什么
- 找到适合自己的 Skill 内容
- 下载可复用模板
- 建立对你个人品牌的认知

## 四、核心价值主张
一个具有品牌质感的个人 Skill 展示网站，让用户既能快速理解你的能力体系，也能直接获取可下载模板与方法内容。

## 五、核心功能模块

### 1. 首页
目标：建立第一印象与品牌感。

建议内容：
- Hero 首屏
- 一句话介绍
- 个人 Skill 能力概览
- 精选 Skill 卡片
- 进入全部 Skill 的导航入口

### 2. Skill 列表页
目标：让用户高效浏览所有 Skill。

建议内容：
- Skill 卡片列表
- 分类/标签（可先保留结构，首版可简化）
- 精选/最新排序（首版可仅做简单排序）

### 3. Skill 详情页
目标：完整承载单个 Skill 的信息表达。

固定字段：
- 标题
- 简介
- 适用场景
- 使用方法
- 可下载模板

建议扩展字段（为后续预留）：
- 标签
- 封面图
- 发布时间
- 版本号
- 关联 Skill

### 4. 关于页
目标：补充个人背景与品牌表达。

建议内容：
- 个人介绍
- 擅长领域
- 内容方向
- 对外合作说明（可后续补充）

## 六、内容模型设计
每个 Skill 应抽象为统一内容对象，建议字段：
- slug
- title
- summary
- scenarios
- howToUse
- download
- category
- tags
- featured
- publishedAt
- locale

首版必须保证前五项可完整展示。

## 七、用户路径
1. 用户进入首页
2. 被品牌视觉和首页精选内容吸引
3. 浏览 Skill 列表
4. 进入某个 Skill 详情页
5. 阅读简介、适用场景、使用方法
6. 下载模板或继续浏览其他 Skill
7. 对你形成能力认知与品牌印象

## 八、非功能要求
- 页面风格高级、简洁、现代
- 内容结构清晰
- 后续新增 Skill 成本低
- 具备 SEO 基础能力
- 兼容后续双语扩展

## 九、首版暂不做
- 后台管理系统
- 用户登录体系
- 在线支付
- 复杂数据统计后台
- 社区互动系统

## 十、MVP 成功标准
- 可以正式对外展示
- 可以快速新增新的 Skill 条目
- 用户可以清晰理解你的能力方向
- 模板下载链路清晰可用
