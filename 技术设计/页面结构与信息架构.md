# 页面结构与信息架构

## 一、目标
围绕桌面 `skills` 目录中的 Skill 资产，构建一个以品牌表达为核心、以结构化内容浏览为主的网站信息架构。

网站首版重点不在复杂功能，而在：
- 第一眼品牌感
- Skill 的结构化展示
- 详情页的专业感与可读性
- 后续可持续新增 Skill

## 二、整体站点结构

/about

## 三、页面职责划分

### 1. 首页 /
目标：建立个人品牌感，并快速引导用户理解你提供的 Skill 能力。

建议模块：
1. 顶部导航
2. Hero 主视觉
3. 一句话价值主张
4. 能力分类概览
5. 精选 Skill
6. 方法论/输出物说明
7. About 引导
8. 页脚

首页不宜过度堆内容，应突出：
- 品牌语气
- 视觉秩序
- 精选内容质量

### 2. Skill 列表页 /skills
目标：让用户高效浏览全部 Skill，并形成能力地图认知。

建议模块：
1. 页面标题与简介
2. 分类切换或分类分区
3. Skill 卡片网格
4. 后续预留标签筛选/搜索

卡片建议展示：
- 标题
- 简介
- 分类
- 标签
- 精选标识（可选）

### 3. Skill 详情页 /skills/[slug]
目标：完整表达单个 Skill 的价值、适用场景和使用方式。

推荐模块顺序：
1. Hero 信息区
2. 适用场景
3. 使用方法
4. 输出内容
5. 详细说明
6. 延伸资料
7. 下载按钮
8. 相关 Skill 推荐（后续可加）

### 4. About 页面 /about
目标：补足人的信息，增强合作信任感。

建议模块：
- 个人介绍
- 擅长方向
- 长期关注主题
- 做事方式 / 方法论
- 可选联系入口

## 四、信息层级设计原则

### 原则 1：首页做“吸引”，列表页做“浏览”，详情页做“说服”
- 首页解决“你是谁、有什么价值”
- 列表页解决“你有哪些 Skill”
- 详情页解决“���个 Skill 具体能做什么、适不适合我”

### 原则 2：内容结构化优先于说明堆砌
从原始 `SKILL.md` 中抽取结构化字段，优先呈现：
- 简介
- 适用场景
- 使用方式
- 输出结果
而不是把所有说明原文无差别堆到页面上。

### 原则 3：品牌感优先于工具感
虽然内容本质是 Skill/工具说明，但前台表达应更像“高质量作品集”，而不是命令行文档中心。

## 五、首页信息架构建议

### Hero 区
文案方向：
- 展示你的能力定位
- 强调方法、产品、AI、工作流等跨领域能力
- 保持高级、克制、不喊口号式堆词

### 精选 Skill 区
建议仅展示 3~4 个，优先挑：
- 代表性强
- 可理解度高
- 能体现你横跨产品、Prompt、工程的能力

### 分类概览区
以分类维度帮助用户建立认知，比如：
- 产品文档生成
- 流程图与工作流
- Prompt 与 AI 交互
- 开发与工程实践
- 测试与效率工具

## 六、详情页内容架构建议
详情页需兼顾“专业”与“可读”。

建议将原始文档拆成以下层级：

### A. 快速理解层
- 标题
- 简介
- 标签 / 分类
- HeroNote

### B. 决策判断层
- 适用场景
- 适合谁用
- 能产出什么

### C. 使用说明层
- 如何触发
- 使用步骤
- 典型输入 / 输出

### D. 深度阅读层
- 原始说明正文
- 延伸资料

## 七、关于视觉与 vibevibe 风格映射
参考 `https://www.vibevibe.cn/zh/` 的方向，网站信息架构要服务于以下气质：
- 留白足
- 模块清晰
- 文案克制
- 色彩少而准
- 强调整体节奏而非局部花哨

这意味着：
- 页面分区不要太碎
- 卡片数量不要在首屏堆太满
- 详情页不要做成技术文档长页面的廉价感

## 八、首版推荐接入范围
建议首版先接入 6~8 个 Skill，保证质量优先。

推荐优先级：
1. ai-companion-prd
2. feature-iteration-prd
3. dify-workflow-builder
4. prompt-optimizer
5. next-best-practices
6. webapp-testing

这些 Skill 覆盖：产品、Prompt、工作流、开发、测试，代表性较强。
